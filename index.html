Decisions log

Top-level architectural commitments. Each is either active or reversed — never "deferred" or "maybe."

STT = Soniox EUrev 1 Committed after direct empirical Hebrew test (3 colloquial samples, near-perfect vs ElevenLabs Scribe v1 catastrophic). Verified HIPAA + SOC 2 Type 2 + ISO 27001 + GDPR + zero retention. EU = Israeli PPL adequacy (Amendment 13). DPA pre-signed, one-click countersign in Console. No sales call needed.
Architecture v4.0 stack: Soniox L0 → L4 narrow correction → Trust Mode UI → Clinical SLP layer → Compliance/auditrev 1 L4 measured-before-locked (no Hebrew empirical evidence narrow LLM correction beats 7.5% baseline). Trust Mode = Abridge Linked Evidence pattern, NOT confidence heatmap (CHI '25 study showed heatmaps don't help).
SLAM-LLM linear projector from-scratch ASRabandoned Trained 9k/10k steps with LoRA r=32 on 50k Hebrew samples. Audio-routing probe: model produces same Hebrew loop for 4 of 5 different audios. WER 614% on FLEURS he_il. Architecture inadequate for non-English at this data scale. Trained model.pt kept locally as historical artifact only.
L4 text-only correction (KenLM, DictaLM rescoring)abandoned KenLM 5-gram shallow fusion regressed on noisy Hebrew. DictaLM-1.7B perplexity rescoring flat. Text-only rescoring of Hebrew N-best = dead. (Note: narrow audio-aware correction of low-conf spans is a different problem, still on the table pending L4 measurement.)
"Israel residency required" framingcorrected PPL Amendment 13 does NOT require physical residency in Israel. It requires one of: adequate destination (EU qualifies), DPA with PPL-aligned clauses, or data subject consent. Soniox EU + DPA + consent satisfies the law. Reversed earlier overcautious framing.
"Use ElevenLabs Scribe because 3.1% Hebrew WER"corrected 3.1% number was fabricated through search-result misattribution and recycled across responses (§7e violation). ElevenLabs' own Hebrew page states 15.2% on FLEURS Hebrew. Reversed in favor of Soniox after direct empirical comparison.

Technical lessons (M-series)

M38 transformers must be imported BEFORE numpy/torch on Windows (DLL load order). Otherwise exit 127 silent abort.
M-jiwer jiwer<3 required — jiwer 3+ breaks truth_transform API used in scripts/eval/metrics.py.
M-datasets datasets<4 required — 4.x needs torchcodec/ffmpeg.
M-numpy numpy<2.1 required for numba compatibility.
M-kenlm-prune KenLM --prune flag values are SEPARATE args, not one quoted string. --prune '0 0 1 1 1' silently fails; correct is --prune 0 0 1 1 1.
M-slam-deepspeed SLAM-LLM checkpoint_handler.py needs deepspeed import wrapped in try/except (CUDA_HOME often absent in containers).
M-slam-encoder SLAM-LLM encoder.py needs getattr(model_config, "whisper_decode", False) instead of struct access (Hydra struct mode rejects unknown keys).
M-slam-join SLAM-LLM Join([model]) needs hasattr(model, "join_hook") check, fallback nullcontext() for single-GPU non-DDP.
M-tf-vs-freerun Token-acc under teacher forcing ≠ working ASR. Free-run probe is required to know if audio is actually being used. Phase 1's 0.38 TF token-acc was LLM learning Hebrew bigram stats, not audio routing.
M-soniox-no-nbest Soniox does NOT expose N-best. L4 correction is "rewrite spans," not "rerank candidates."
M-soniox-cap Soniox custom-vocabulary context capped at 8,000 tokens (~10K chars).
M-soniox-conf Soniox word confidence is 0.0–1.0 but NOT calibrated. Threshold must be empirically tuned on Hebrew before commit.
M-eu-adequacy EU has Israeli adequacy decision under PPL Amendment 13; US does not. Use api.eu.soniox.com + DPA + patient consent.
M-aphasiabank AphasiaBank has ZERO Hebrew corpus (verified talkbank.org). We build it.
M-elevenlabs-he ElevenLabs Scribe v1 Hebrew = 15.2% WER (per elevenlabs.io/speech-to-text/hebrew), NOT 3.1%. The 3.1% number recycled across prior sessions was fabricated through search-result misattribution.
M-dictalm-3-sizes DictaLM 3.0 line = 1.7B / 12B / 24B only. There is no 7B in 3.0; older 2.0 line had 7B.
M-chi25-conf CHI '25 study: confidence heatmaps do NOT improve user correction efficiency. Use Linked Evidence (Abridge) pattern instead.
M-l4-hebrew No published Hebrew empirical evidence that narrow LLM correction improves a 7.5% ASR baseline. Must measure before locking L4 in.
M-recompute §7e: don't recycle numbers across configs. The 3.1% recycle was the canonical violation.
